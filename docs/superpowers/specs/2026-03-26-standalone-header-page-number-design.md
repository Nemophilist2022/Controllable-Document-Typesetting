# 单独改页码与单独改页眉设计

## 目标

在现有“单独设置”面板中新增两个局部处理功能：

- 单独改页码
- 单独改页眉

这两个功能都采用严格模式：只在现有分节上改页码或页眉，不主动补分节，最大化少动正文。

## 背景与现状

当前项目已经支持两类局部处理模式：

- `toc.only_insert`：单独插入或更新目录
- `cover.only_insert`：单独插入外部封面

同时，页码和页眉本身已有独立处理函数：

- `setup_page_numbers(doc, cfg)`
- `setup_headers(doc, cfg)`

这说明“单独改页码”和“单独改页眉”可以沿用现有局部模式架构，而不需要新增一套独立入口。

## 设计决策

### 1. 新增两个局部模式

在配置中新增两个布尔项，默认值均为 `false`：

- `page_numbers.only_insert`
- `header_footer.only_insert`

这两个模式与现有的 `toc.only_insert`、`cover.only_insert` 同级。

### 2. 运行模式互斥

局部处理模式统一互斥：

- 单独插入/更新目录
- 单独插入外部封面
- 单独改页码
- 单独改页眉

若同时启用多个局部模式，则直接报错，不做隐式优先级处理。

### 3. 严格模式边界

#### 单独改页码

- 只基于现有 section 修改页码格式、起始值、页脚页码位置。
- 不主动插入分节符。
- 不按正文起点补 section。
- 不修改正文、标题、目录、题注、封面、参考文献等其他内容。

#### 单独改页眉

- 只基于现有 section 清理和重建页眉。
- 不主动插入分节符。
- 不按正文起点补 section。
- 不修改正文、标题、目录、题注、封面、参考文献等其他内容。

### 4. 用户可见限制

严格模式必须明确提示以下限制：

- 如果原文本来只有一个分节，或者前置部分与正文没有按分节拆开，那么“仅正文页眉/页码”的效果只能按现有分节结构生效。
- 工具不会在这两个局部模式下自动修正前置部分与正文的边界。
- 这属于功能定义的一部分，不视为缺陷。

## GUI 设计

### 1. 单独设置面板新增两个选项

“单独设置”面板保留并展示 4 个单独处理项：

- 单独插入/更新目录（不改其他格式）
- 单独插入外部封面（不改正文与其他格式）
- 单独改页码
- 单独改页眉

### 2. 交互规则

- 这 4 个选项互斥，勾选其一时自动取消另外 3 个。
- 勾选任一单独处理项时，同时取消顶部“跳过目录生成”，避免语义冲突。
- “单独改页码”和“单独改页眉”不新增独立参数区，继续复用“页眉页码”面板里已有的配置项。

### 3. 备注文案

备注必须使用纯中文，不出现英文。建议文案：

- 单独改页码：`只按现有分节调整页码，不自动补分节，尽量少动正文。`
- 单独改页眉：`只按现有分节调整页眉，不自动补分节，尽量少动正文。`

## 执行流程设计

### 1. apply_format 入口统一判断局部模式

在 `apply_format()` 中统一判断以下模式：

- `toc.only_insert`
- `cover.only_insert`
- `page_numbers.only_insert`
- `header_footer.only_insert`

命中局部模式时，直接进入对应的最小处理分支，而不是继续走完整格式化流程。

### 2. 单独改页码分支

处理步骤：

1. 打开原始 `Document(input_path)`。
2. 直接调用“严格页码更新”逻辑。
3. 仅在现有 section 上更新页码格式、页脚内容和位置。
4. 保存到输出文件。

这个分支不得调用会新增 section 的逻辑，例如：

- `insert_section_break_before()`
- 按正文起点重建分节边界的流程

### 3. 单独改页眉分支

处理步骤：

1. 打开原始 `Document(input_path)`。
2. 直接调用“严格页眉更新”逻辑。
3. 仅在现有 section 上清理并写入页眉。
4. 保存到输出文件。

这个分支同样不得补分节。

## 后处理策略

为避免“单独改页码/页眉”顺带刷新目录，后处理应细分为三档：

- `full`：完整后处理，更新目录和域
- `fields_only`：只更新域，不处理目录样式
- `none`：不做 Word 后处理

推荐规则：

- `cover.only_insert` -> `none`
- `toc.only_insert` -> `full`
- `page_numbers.only_insert` -> `none`
- `header_footer.only_insert`：
  - 如果页眉文本包含章节标题引用占位，则使用 `fields_only`
  - 否则使用 `none`

这样可以保证：

- 单独改页码不会顺带动目录
- 单独改页眉只有在确实需要更新章节标题域时才做最小后处理

## 日志设计

启动后日志建议如下：

- 单独改页码：`单独处理：仅更新页码`
- 单独改页眉：`单独处理：仅更新页眉`

如触发限制场景，不额外报错，只按现有分节结构生效。

## 测试设计

建议新增或补充以下自动化测试：

### 1. GUI 互斥测试

验证“单独设置”4 个选项互斥，并且勾选任一项时会清除顶部“跳过目录生成”。

### 2. 页码严格模式测试

验证：

- 不新增 section
- 只修改现有页脚与页码格式
- 不触发完整格式化流程

### 3. 页眉严格模式测试

验证：

- 不新增 section
- 只修改现有页眉
- 在“仅正文”范围下，仍按现有 section 边界生效

### 4. 后处理策略测试

验证：

- `page_numbers.only_insert` 不跑 Word 后处理
- `header_footer.only_insert` 在普通静态页眉下不跑后处理
- `header_footer.only_insert` 在章节标题引用场景下只跑最小域更新

## 推荐实现顺序

1. 在默认配置中新增两个 `only_insert` 开关。
2. 在 GUI “单独设置”里增加两个复选项与纯中文备注。
3. 补齐 4 个局部模式的统一互斥逻辑。
4. 在 `apply_format()` 中增加两个严格模式分支。
5. 为后处理引入更细的执行策略。
6. 补自动化测试并回归现有测试。

## 非目标

本次设计不包含：

- 自动修正文前置部分与正文的分节边界
- 在单独改页码/页眉时顺带修复目录、题注、标题等其他内容
- 新增额外命令行参数入口
